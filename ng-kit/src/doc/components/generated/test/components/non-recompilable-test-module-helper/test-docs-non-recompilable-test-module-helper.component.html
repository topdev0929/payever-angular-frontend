If you detected that your test cases (inside it('...') block) has too long execution time - 1 or more seconds - usually it happens because of long module recomplication by TestBed.configureTestingModule() & await TestBed.compileComponents().

One of better approaches here is to prevent Angular' TestBed full module recompilation per each spec file using this helper (see examples).


NOTE #1: This module doesn't works well with Angular Router

HACK NOTE: Use karma-spec-reporter to measure tests execution time (but it not works with PhantomJS).
